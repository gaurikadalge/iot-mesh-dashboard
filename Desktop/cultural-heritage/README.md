# Cultural Heritage Site Management System

A DBMS project demonstrating **Polyglot Persistence** by using:
- **PostgreSQL** for structured heritage site data
- **MongoDB** for unstructured oral histories
- **FastAPI** as backend with a simple dashboard

## Overview
This project manages data related to cultural heritage sites, monuments, visitors, and historical records. It applies core DBMS concepts while using different databases based on data structure and storage requirements.

## Databases Used
- PostgreSQL: Heritage sites, monuments, events (structured data)
- MongoDB: Oral histories, narratives (unstructured data)

## Technologies
- PostgreSQL
- MongoDB
- FastAPI (Python)
- HTML, CSS (Tailwind)

## DBMS Concepts Applied
- Schema design
- Primary and foreign keys
- Normalization
- Relational and NoSQL databases
- CRUD operations

## Objective
To design an efficient database system for cultural heritage management using appropriate database technologies.
